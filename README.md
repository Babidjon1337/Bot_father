# Bot Father SaaS

## 🛠 Запуск проекта

1. **Активация виртуального окружения:**
   ```bash
   .\venv\Scripts\activate
   ```
2. **Запуск основного приложения:**
   ```bash
   python main.py
   ```

## 🗄 Работа с базой данных (Alembic)

В проекте используется **Alembic** для управления структурой базы данных (миграциями). 

### 1. Создание новой миграции
Если ты изменил `database/models.py` (добавил колонку, таблицу, изменил тип данных), нужно сгенерировать миграцию:
```bash
python -m alembic revision --autogenerate -m "add_new_column_name"
```
*Alembic сам сравнит твои классы из кода с реальными таблицами в БД и создаст файл с инструкциями в папке `database/migrations/versions/`.*

### 2. Применение миграций (Обновление БД)
Чтобы применить созданную миграцию и реально обновить таблицы:
```bash
python -m alembic upgrade head
```
*`head` означает "накатить все новые миграции до самой последней". Эту же команду нужно выполнять на сервере при деплое.*

### 3. Откат изменений (Downgrade)
Если ты сделал миграцию по ошибке и хочешь вернуть базу на один шаг назад (отменить последнее изменение):
```bash
python -m alembic downgrade -1
```

### 💡 Возможные ошибки
- **alembic: command not found**: Убедись, что активировано виртуальное окружение (`venv`) или запускай через `python -m alembic ...`.
- **Target database is not up to date**: Ты пытаешься создать новую миграцию, но забыл применить старые. Сначала выполни `python -m alembic upgrade head`.
