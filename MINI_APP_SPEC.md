# Спецификация Telegram Mini App: Bot Father Dashboard

Этот документ описывает визуальную концепцию, пользовательский опыт и техническую архитектуру Mini App для конструктора ботов.

---

## 1. Визуальная концепция: "Native & Fluid"

**Общий стиль:** Приложение должно ощущаться как органическое продолжение Telegram. Мы используем библиотеку **TGUI (Telegram UI)**.
- **Цветовая палитра:** Полная адаптация под системную тему пользователя (`--tg-theme-bg-color`, `--tg-theme-button-color`). В темной теме — глубокий графитовый, в светлой — чистый белый с мягкими тенями.
- **Тактильность:** Каждое действие сопровождается Haptic Feedback (легкая вибрация при сохранении, уведомление при ошибке).
- **Анимации:** Плавные переходы между списком блоков и схемой. Сообщения "всплывают" при добавлении, как в чате.

---

## 2. Путешествие пользователя (User Journey)

### Экран А: Onboarding & Subscription (Замок)
Представь чистый экран с иконкой робота-отца. 
1. **Инструкция:** Короткий горизонтальный слайдер (3 шага): "Создай воронку" -> "Подключи бота" -> "Получай прибыль".
2. **Ввод Email:** Единственное текстовое поле для чеков.
3. **Оплата:** Большая кнопка внизу (Main Button) с текстом "Подписаться за 1500₽/мес". 
   - *Эффект:* После нажатия открывается нативное окно оплаты. При успешной транзакции экран "разлетается" конфетти, открывая доступ к конструктору.

### Экран Б: Редактор блоков (Рабочий стол)
Это вертикальная лента карточек. Каждая карточка — это один шаг (Node) воронки.
- **Карточка сообщения:** 
  - Сверху номер шага и его ID (например, "Старт" или "Дожим 1").
  - Поле для текста с поддержкой Markdown (жирный, курсив подсвечиваются сразу).
  - Слот для медиа: если файл загружен, показывается миниатюра (превью).
  - **Кнопка:** Всего одна. Можно задать текст и выбрать тип (Оплата / Ссылка / Переход к шагу).
  - **Таймер:** Небольшой значок часов в углу, где настраивается задержка перед следующим "пауком".

### Экран В: Визуальная схема (The Web)
По нажатию на иконку "паутины" в шапке, список блоков превращается в интерактивную карту.
- **Вид:** Ноды выглядят как маленькие облачка сообщений.
- **Связи:** Из "хвоста" одного облачка тянется линия к другому. Если это дожим — линия пунктирная. Если переход по кнопке — линия сплошная.
- **Интерактив:** Можно зумить карту пальцами. Тап по облачку — и ты мгновенно возвращаешься в редактор именно к этому блоку.

### Экран Г: Статистика (Пульс бизнеса)
Простая панель с цифрами:
- **Banner:** Общая сумма заработка за месяц в золотистой рамке.
- **Воронка:** Вертикальная гистограмма. На каждом уровне написано: "Просмотрели: 500", "Кликнули: 200", "Купили: 50". Сразу видно, где "дыра" в продажах.

## 3. Техническая архитектура и общение (Bridge)

Mini App (React) и Backend (FastAPI) общаются как два старых друга через защищенный мост.

### Безопасность (The Handshake)
При каждом запросе фронтенд передает `window.Telegram.WebApp.initData`. 
Бэкенд берет эту строку, проверяет её HMAC-подпись с помощью `BOT_TOKEN`. Если подпись верна — мы точно знаем, кто этот пользователь, и что его данные не подделаны.

### Обмен данными (The JSON Flow)
Бэкенд отдает фронтенду один большой JSON объект воронки:
```json
{
  "is_subscribed": true,
  "stats": { "total_earned": 15000, "conversion": 12.5 },
  "funnel": { ... }
}
```
Фронтенд хранит это состояние в **React Context** или **Zustand**. Любое изменение в блоке сразу отправляет `PATCH` запрос на бэкенд для автосохранения.

### Работа с медиа (Media Pipeline)
1. Пользователь выбирает фото в Mini App.
2. Файл летит на эндпоинт `/api/upload`.
3. Бэкенд сохраняет его в облако (S3) или локально и возвращает временную ссылку.
4. Когда пользователь нажимает "Опубликовать", бот отправляет этот файл сам себе, получает `file_id` и сохраняет его в схему воронки.

### Рекуррентные платежи (Money Engine)
- Бэкенд хранит `yookassa_payment_method_id` после первой оплаты.
- Раз в 30 дней срабатывает фоновая задача. Если списание успешно — поле `subscription_until` в БД продлевается. Если нет — лиду отправляется уведомление в бот "Твоя подписка истекла".

## 4. Стек технологий (Stack)
- **Frontend:** React + Vite + TypeScript.
- **Canvas:** React Flow (для схемы паутины).
- **UI:** @twa-dev/sdk + Telegram UI components.
- **Backend API:** FastAPI (Python).
- **Database:** PostgreSQL (SQLAlchemy).
- **Async Tasks:** APScheduler (для рекуррентных списаний).
